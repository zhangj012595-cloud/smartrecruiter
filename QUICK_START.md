# SmartRecruiter v2.0 快速启动指南

## 🚀 一分钟快速开始

### 步骤1：进入项目目录
```bash
cd smartrecruiter-mvp
```

### 步骤2：安装依赖（如果尚未安装）
```bash
pip install -r requirements.txt
```

### 步骤3：启动增强版应用
```bash
streamlit run app_enhanced.py
```

### 步骤4：在浏览器中访问
打开浏览器，访问：`http://localhost:8501`

## 🎯 核心功能快速体验

### 1. **使用GitHub渠道搜索开发者**（立即可用，完全免费）

**操作步骤：**
1. 在左侧边栏确认GitHub渠道已启用（默认✅启用）
2. 切换到【职位描述】标签页
3. 输入职位描述，点击"分析职位描述"
4. 切换到【候选人搜索】标签页
5. 选择数据源：`github`
6. 设置搜索条件（可选）：
   - 地理位置：如 "China"
   - 最小关注者：10
   - 最小仓库数：5
7. 点击"🚀 开始搜索"

**预期结果：**
- 搜索GitHub上的开发者
- 按项目质量和社区影响力评分
- 显示统一格式的候选人卡片

### 2. **使用LinkedIn渠道搜索职场人士**（需要认证）

**操作步骤：**
1. 在左侧边栏的LinkedIn配置区域输入账号密码
2. 确保LinkedIn渠道已启用
3. 在【候选人搜索】标签页选择数据源：`linkedin`
4. 设置工作地点和经验级别
5. 点击开始搜索

### 3. **使用模拟数据测试**（无需认证）

**操作步骤：**
1. 选择数据源：`mock_linkedin`
2. 直接点击开始搜索
3. 查看模拟的候选人数据

## ⚙️ 配置管理快速指南

### 渠道管理
1. **查看渠道状态**：左侧边栏显示所有渠道状态
2. **启用/禁用渠道**：点击渠道旁边的"切换"按钮
3. **编辑渠道配置**：在【系统配置】→【渠道配置】标签页

### GitHub配置
- **无需配置**：默认即可使用
- **可选配置**：在侧边栏输入GitHub Token可提高API限制

### 搜索配置
- **默认搜索数量**：20个候选人
- **地点配置**：在【系统配置】→【搜索配置】添加常用地点

## 🐛 常见问题快速解决

### 问题：GitHub搜索返回"未找到候选人"
**解决方案：**
1. 放宽搜索条件（降低关注者和仓库数要求）
2. 移除地理位置限制
3. 确保网络连接正常

### 问题：ModuleNotFoundError: No module named 'requests'
**解决方案：**
```bash
pip install requests
# 或重新安装所有依赖
pip install -r requirements.txt
```

### 问题：Streamlit应用启动失败
**解决方案：**
1. 检查是否在正确的目录：`smartrecruiter-mvp/`
2. 检查端口占用：`lsof -ti:8501`
3. 尝试其他端口：`streamlit run app_enhanced.py --server.port 8502`

## 💡 使用技巧

### 高效搜索技巧：
1. **关键词精确**：使用具体的技能关键词
2. **条件适度**：不要设置过于严格的条件
3. **分步搜索**：先宽后窄，逐步筛选

### 候选人评估技巧：
1. **关注项目质量**：查看高星标项目
2. **检查社区影响力**：关注者数量和项目参与度
3. **评估技术深度**：查看技能分布和项目复杂度

### 配置管理技巧：
1. **定期备份配置**：导出配置到JSON文件
2. **测试新渠道**：先使用Mock数据测试
3. **渐进启用**：一次只启用一个渠道进行测试
