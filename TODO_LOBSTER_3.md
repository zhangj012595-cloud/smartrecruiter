# 🦞 TODO龙虾3：工作流程系统性优化

## 🎯 **问题描述**

### **症状**
AI助手在会话启动时容易丢失项目上下文，错误地在工作空间根目录创建新项目，而不是在现有项目中继续迭代。

### **具体案例**
**日期**: 2026-04-16  
**项目**: SmartRecruiter (hr-2项目)  
**错误**: 在工作空间根目录创建了新的招聘配置系统项目  
**正确位置**: `smartrecruiter-mvp/` 目录  

### **影响**
1. **重复工作**: 在错误位置开发功能，需要重新迁移
2. **项目混淆**: 多个相似项目并存，难以管理
3. **效率降低**: 需要手动纠正和重新整合
4. **用户体验**: 用户需要不断提醒AI助手正确位置

---

## 🔍 **根本原因分析**

### **技术原因**
1. **会话启动流程缺陷**
   - 启动新会话时未检查现有项目结构
   - 没有项目上下文记忆机制
   - 缺乏工作目录验证

2. **工作空间认知偏差**
   - 将工作空间根目录视为"默认"工作区
   - 没有意识到应该在特定项目目录中工作
   - 缺乏项目切换意识

3. **缺乏标准化检查**
   - 没有项目上下文检查清单
   - 没有自动识别活跃项目的机制
   - 缺乏用户确认流程

### **流程原因**
1. **开发模式单一**
   - 总是假设"从零开始"的开发模式
   - 缺乏"继续开发"的标准化流程
   - 没有项目继承机制

2. **上下文传递失败**
   - 会话间项目状态不传递
   - 用户意图理解不准确
   - 缺乏项目历史追踪

---

## 🎯 **优化目标**

### **主要目标**
1. **零错误启动**: 确保每次会话都在正确的项目上下文中启动
2. **智能上下文识别**: 自动识别用户意图和项目上下文
3. **无缝继续开发**: 支持从上次中断处继续开发

### **具体指标**
1. **错误率降低**: 项目上下文错误减少90%以上
2. **启动时间**: 会话启动时间增加不超过5秒
3. **用户满意度**: 减少用户需要手动纠正的频率
4. **开发效率**: 提高继续开发的效率

---

## 🛠️ **优化方案**

### **阶段1：增强会话启动流程** (立即实施)

#### **1.1 项目上下文检查清单**
```python
# PROJECT_CONTEXT_CHECKLIST.md 标准化流程
1. 检查当前工作目录
2. 扫描工作空间中的项目目录
3. 识别最近活跃的项目
4. 检查项目状态（Git、文件修改时间）
5. 提示用户确认工作目录
6. 记录项目上下文到会话状态
```

#### **1.2 自动项目识别**
- **特征识别**: 通过项目文件识别项目类型
- **活跃度排序**: 按最近修改时间排序项目
- **用户偏好**: 记录用户常访问的项目

#### **1.3 用户确认机制**
```
[系统提示] 检测到以下活跃项目：
1. smartrecruiter-mvp/ (最近修改: 2小时前)
2. recruitment-config-system/ (最近修改: 5分钟前)

请确认工作目录：
1. 继续 smartrecruiter-mvp/
2. 切换到 recruitment-config-system/
3. 创建新项目
4. 其他（手动指定）
```

### **阶段2：项目上下文记忆** (短期实施)

#### **2.1 项目状态记录**
```json
// memory/project_context.json
{

    }
  },

    "default_project": "smartrecruiter-mvp",
    "recent_projects": ["smartrecruiter-mvp", "recruitment-config-system"]
  }
}
```

#### **2.2 会话间状态传递**
- **项目锁定**: 记录当前会话的工作项目
- **状态同步**: 在会话间同步项目状态
- **历史追踪**: 记录项目访问历史

#### **2.3 智能上下文恢复**
- **自动恢复**: 基于对话历史恢复项目上下文
- **意图识别**: 分析用户请求中的项目线索
- **上下文验证**: 验证恢复的上下文准确性

### **阶段3：工作流程标准化** (中期实施)

#### **3.1 标准化开发模式**
```
模式1: 继续开发 (Continue)
   - 在现有项目基础上继续
   - 继承所有配置和状态
   - 支持增量开发

模式2: 新功能开发 (Feature)
   - 在现有项目中添加新功能
   - 保持项目架构兼容性
   - 模块化集成

模式3: 新项目创建 (New)
   - 明确创建新项目
   - 独立的工作目录
   - 完整的新项目流程
```

#### **3.2 项目切换工具**
```bash
# 项目切换命令
project switch smartrecruiter-mvp
project list  # 列出所有项目
project status  # 显示当前项目状态
project create new-project  # 创建新项目
```

#### **3.3 开发状态保存**
- **检查点**: 定期保存开发状态
- **快照**: 创建项目状态快照
- **恢复点**: 支持从检查点恢复
